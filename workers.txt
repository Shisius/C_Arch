есть Объект worker
есть диспетчер Dispatcher
есть задача (Task) - некая функция, обернутая в класс.
Task умеет:
	1. Измерять время выполнения функции
	2. Мониторить загруженность процессора
  3. 
Worker умеет:
	1. имеет указатель на dispatcher
	2. запускать поток с задачей Task
  3. посылать сообщения to dispatcher
  4. обладает буферами обмена для входящих и исходящих данных.
  
Dispatcher умеет:
  1. Создавать и уничтожать workers.
  
  
Тест:
  1. На COM порту - GNSS датчик. Один поток непрерывно принимает данные с него.
  2. Второй поток на в N милисекунд генерит матрицу MxN.
  3. Третий поток берёт сгенерированные данные у первого и второго и генерирует свои данные.
  4. Каждая генерация данных - некая заменяемая функция - Task.
  5. Четвертый поток передает данные третьего по Ethernet.
  6. Каждый поток работает со своей частотой.
  7. Для данных должен быть флаг обновления.
